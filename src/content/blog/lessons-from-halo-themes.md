浏览 Halo 的主题市场时，很容易先被封面图、动效和配色吸引。但真正决定一个博客能否长期使用的，往往是更安静的部分。

## 一、首页不是文章卡片仓库

许多博客把每篇文章放进完全相同的卡片里。它很整齐，却会让标题、摘要和图片拥有同样的音量，读者很难判断从哪里开始。

更好的主题会建立节奏：

- 一篇精选文章获得更大的视觉空间；
- 最近更新使用轻量的列表，方便快速扫描；
- 分类只在需要时出现，而不是变成装饰标签；
- 留白负责分组，不必为每个内容区再套一层圆角容器。

Lumos 的首页因此只保留一个强烈的海报框，其余文章回到开放的横向列表。视觉焦点更少，阅读入口反而更明确。

## 二、正文排版比首页更重要

首页决定第一次印象，正文决定读者是否愿意留下。

中文长文尤其依赖合适的行宽与行高。行太长，视线回到下一行时容易迷失；行太短，又会让段落频繁换行。这个主题把正文限制在约 720 像素，并使用接近两倍字号的行高。

标题、引用、代码与表格也需要各自清晰，但不抢夺正文注意力。设计目标不是“每一种元素都好看”，而是让它们共同服务于连续阅读。

## 三、把风格写成变量

一套主题能否被维护，取决于设计决策是否集中。

```css
:root {
  --paper: #f5f2e9;
  --ink: #171714;
  --muted: #6d695f;
  --light: #ddff57;
  --line: rgba(23, 23, 20, 0.2);
}
```

背景、文字、分隔线与强调色使用语义化变量后，暗色模式不需要重写每一个组件；日后调整品牌气质，也不必在几十个文件里寻找颜色值。

这也是成熟主题最值得学习的地方：它不仅提供一个当下好看的结果，还为未来的变化留下结构。

## 少一点模板感

主题应该帮助内容形成性格，而不是把所有站点塑造成同一张模板。删掉没有信息价值的徽章、统计数字和装饰图标，选择一两个真正属于自己的视觉动作，通常就足够了。

对于 Lumos，那两个动作是纸张与墨色的对比，以及在深色画面里亮起的柠檬黄色光球。其余部分尽量安静。
