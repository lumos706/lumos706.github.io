# 旅游攻略系统：视觉与交互锁定

## 设计命题

这不是一张“景点卡片墙”，而是一册四人西北出行的公路档案。页面沿用 Lumos 的暖纸、黑墨、宋体标题与荧光色，只把博客的“阅读海报”换成带地形等高线、路径节点和里程刻度的路线板。

`ui-ux-pro-max` 返回的玻璃拟态、蓝橙 SaaS 配色与本项目既有风格和西北旅行主题都不匹配，因此不采用。保留它对触控尺寸、明确选中状态、移动端不依赖 hover、可见焦点和 reduced motion 的约束。

## 单一记忆点

右侧黑色路线板：一条从沈阳到西宁、再穿过青海与河西走廊的荧光路径，既是首屏主视觉，也是当前方案的路线摘要。页面其他区域保持克制。

## 设计令牌

- 纸张：`#f5f2e9`
- 深纸：`#ebe5d8`
- 墨色：`#171714`
- 次级文字：`#6d695f`
- 荧光路线：`#ddff57`
- 青海蓝：`#4f7f8d`
- 丹霞赭：`#b85f3c`
- 辅助紫：`#6657d8`
- 标题：Georgia + 中文宋体回退
- 界面文字：系统无衬线字体栈
- 几何：直角、细线、开放分栏；不使用玻璃卡片、圆角胶囊或默认 bento 网格
- 动效：路线节点与内容轻微进入，`180–520ms`；系统减少动态时完全禁用

## 信息架构

1. Lumos 全站导航（新增“旅行”）
2. 首屏路线板与明确的行程背景
3. 天数、落地预算两组切换控件
4. 当前路线摘要与按日行程
5. 预算账本与档位差额
6. 每日公开价格快照（机票、住宿、门票）
7. 小红书、抖音等公开攻略线索与避坑提示
8. 数据来源、更新时间、无 API 模式说明

## 首屏文案锁定

- 标题：`四个人，向西北。`
- 说明：`从沈阳桃仙出发，在青海与甘肃之间选择一条刚好走得完的路线。先定天数，再定落地预算。`
- 出行条件：`2026 年 8 月初 / 4 位成人 / 2 间双床房 / 租车 + 公共交通`
- 控件：`3 日`、`4 日`、`7 日`；`¥2,500`、`¥3,000`、`¥3,500`
- 预算口径：`落地预算 / 人，不含沈阳往返机票`
- 主操作：`查看当前路线`

## 交互锁定

- 默认：`7 日 + ¥3,500`
- 天数或预算切换后，路线标题、站点、每日安排、费用总额、档位提示与 URL 查询参数同步更新
- 页面刷新或分享带查询参数的 URL 时恢复同一方案
- 所有切换按钮至少 `44px` 高，键盘可操作，使用 `aria-pressed`
- 报价缺乏可靠公开证据时明确显示“暂无可靠公开报价”，禁止伪造数字
- 数据超过 36 小时标记“数据可能过期”；采集失败时继续保留上一次成功快照

## 可拆卸边界
